## QUICK START GUIDE (5 MINUTES)

### Step 1: Create Project Directory
```bash
cd d:\ai
npm create vite@latest StudyFlowAI -- --template react
cd StudyFlowAI
```

### Step 2: Install All Dependencies
```bash
npm install
npm install tailwindcss postcss autoprefixer framer-motion recharts lucide-react react-hot-toast firebase axios clsx react-router-dom
npx tailwindcss init -p
```

### Step 3: Copy All Source Code Files
Copy all the code from STUDYFLOW-COMPLETE-CODE-PART1.txt through PART4.txt into:
- src/components/
- src/pages/
- src/context/
- src/firebase/
- src/services/
- src/hooks/

### Step 4: Setup Environment
Create `.env` file in project root:
```
VITE_FIREBASE_API_KEY=your_key_here
VITE_FIREBASE_AUTH_DOMAIN=your_domain_here
VITE_FIREBASE_PROJECT_ID=your_project_here
VITE_FIREBASE_STORAGE_BUCKET=your_bucket_here
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id_here
VITE_FIREBASE_APP_ID=your_app_id_here
VITE_GEMINI_API_KEY=your_gemini_key_here
```

### Step 5: Run Dev Server
```bash
npm run dev
```

App opens at http://localhost:3000

## DETAILED SETUP INSTRUCTIONS

### Firebase Setup (Free Tier)

1. Go to https://console.firebase.google.com/
2. Click "Create Project"
3. Fill in project name "StudyFlowAI"
4. Continue with default settings
5. Once created, click on your project

#### Enable Authentication:
- Left sidebar → Authentication
- Click "Get Started"
- Enable "Email/Password"
- Enable "Google"
- Click "Web" configuration
- Copy credentials to .env

#### Setup Firestore:
- Left sidebar → Firestore Database
- Click "Create Database"
- Select "Start in test mode" (for development)
- Select region (closest to you)
- Create

#### Add Security Rules:
In Firestore → Rules tab, replace with:

```firestore
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Allow read/write for authenticated users
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

### Google Gemini API Setup

1. Visit https://aistudio.google.com/app/apikey
2. Click "Get API Key"
3. Click "Create API Key"
4. Copy the key
5. Add to .env file as `VITE_GEMINI_API_KEY=xxx`

## FEATURES DEEP DIVE

### 1. AI Study Schedule Generator
**Location**: StudyPlannerPage

How it works:
1. User inputs subjects, exam dates, weak topics, daily hours
2. Form sends to Gemini API via geminiService.js
3. AI generates personalized 7-day schedule
4. Schedule saved to Firestore under studyPlans collection
5. User can view and refine the plan

API Call:
```javascript
const schedule = await generateStudySchedule(
  ['Math', 'Physics'],
  ['Jan 15', 'Feb 1'],
  ['Calculus', 'Optics'],
  5
)
```

### 2. AI Chat Assistant
**Location**: AIAssistantPage

How it works:
1. User types question
2. Message sent to Gemini API
3. AI responds with study advice
4. Chat history saved to Firestore
5. Suggested prompts help users get started

Features:
- Message history loading
- Real-time typing feedback
- Auto-scroll to latest
- Suggested starter prompts

### 3. Pomodoro Timer
**Location**: DashboardPage, usePomodoroTimer hook

Features:
- 25-minute focus sessions
- 5-minute breaks
- Session counter
- Sound notification
- Pause/Resume/Reset controls

Hook provides:
- minutes, seconds (display)
- isActive (running state)
- isBreak (break vs work)
- sessions (count)
- toggle(), reset() (controls)

### 4. Habit Tracker
**Location**: HabitTrackerPage

Features:
- Create custom habits
- Daily completion checkmarks
- Automatic streak calculation
- Persistent storage in Firestore
- Delete habits option

Data structure:
```javascript
{
  id: "habit1",
  name: "Morning Meditation",
  frequency: "daily",
  completed: ["2024-01-15", "2024-01-16"],
  streak: 2,
  createdAt: timestamp
}
```

### 5. Analytics Dashboard
**Location**: AnalyticsPage

Charts:
- **Line Chart**: Weekly study hours trend
- **Bar Chart**: Weekly productivity scores
- **Pie Chart**: Time spent per subject

Stats shown:
- Total study hours (this week/month)
- Productivity percentage
- Current streak days
- Number of subjects studied

### 6. Real-time Firestore Sync
**Location**: firestoreService.js

Collections:
- **users**: User profiles
- **tasks**: To-do items
- **habits**: Tracked habits
- **studyPlans**: Generated schedules
- **chats**: AI chat history
- **analytics**: User metrics

CRUD Operations:
```javascript
// Create
const id = await createTask(userId, {title, dueDate, completed})

// Read
const tasks = await getUserTasks(userId)

// Update
await updateTask(taskId, {completed: true})

// Delete
await deleteTask(taskId)
```

## CUSTOMIZATION GUIDE

### Change Color Scheme

**File**: tailwind.config.js

```javascript
colors: {
  primary: '#6366f1',      // Purple
  accent: '#22c55e',       // Green
  dark: '#0f172a',         // Dark blue
  darkCard: '#1e293b',
  darkInput: '#334155'
}
```

Change these hex values to customize colors throughout the app.

### Add New Pages

1. Create file in `src/pages/NewPage.jsx`
2. Import in `src/App.jsx`
3. Add route:
```javascript
<Route path="/new-page" element={<ProtectedRoute><NewPage /></ProtectedRoute>} />
```
4. Add to sidebar menu in `src/components/Sidebar.jsx`

### Add New Features

Example: Add XP Points System

```javascript
// In firestoreService.js
export const updateXP = async (userId, points) => {
  const analytics = await getUserAnalytics(userId)
  const newXP = (analytics?.xp || 0) + points
  await updateAnalytics(userId, { xp: newXP })
}

// In components where user completes action
toast.success(`+${points} XP!`)
await updateXP(user.uid, points)
```

## DEPLOYMENT

### Deploy to Vercel

1. Push code to GitHub
2. Go to https://vercel.com
3. Click "New Project"
4. Import your GitHub repository
5. Add environment variables:
   - VITE_FIREBASE_API_KEY
   - VITE_FIREBASE_AUTH_DOMAIN
   - etc.
6. Click Deploy

**Deployment takes 1-2 minutes**

### Deploy to Firebase Hosting

```bash
# Build first
npm run build

# Install Firebase CLI
npm install -g firebase-tools

# Login to Firebase
firebase login

# Initialize Firebase
firebase init hosting

# Deploy
firebase deploy
```

### Deploy to Netlify

1. Build: `npm run build`
2. Go to https://netlify.com
3. Drag & drop `dist` folder
4. Add environment variables in settings
5. Done!

## TROUBLESHOOTING

### Issue: Firebase credentials not working
**Solution**: 
- Verify .env file exists in root
- Check credentials in Firebase Console
- Restart dev server after changing .env

### Issue: Gemini API errors
**Solution**:
- Ensure API key is valid
- Check quota at Google AI Studio
- Try refreshing the key

### Issue: Components not loading
**Solution**:
- Check import paths
- Verify all files are created
- Run `npm install` again

### Issue: Styling looks broken
**Solution**:
- Ensure Tailwind config is correct
- Run `npm run build` to check
- Clear browser cache

### Issue: Authentication not working
**Solution**:
- Check Firebase auth providers enabled
- Verify OAuth redirect URLs
- Check browser console for errors

## PERFORMANCE TIPS

1. **Lazy load routes**:
```javascript
const DashboardPage = lazy(() => import('./pages/DashboardPage'))
```

2. **Memoize components**:
```javascript
export const Card = memo(function Card({children}) {...})
```

3. **Optimize images**: Compress all images to <100KB

4. **Use React DevTools**: Install extension to find bottlenecks

5. **Enable code splitting**: Vite does this automatically

## SECURITY BEST PRACTICES

1. **Never commit .env file** - Add to .gitignore ✓
2. **Use HTTPS only** - Vercel/Firebase does automatically
3. **Validate input** - Check form inputs before sending
4. **Sanitize output** - React auto-escapes XSS
5. **Use CORS properly** - Firebase handles this
6. **Rate limit API calls** - Add delays if needed
7. **Secure Firestore rules** - Only authenticated users
